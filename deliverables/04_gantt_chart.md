# Project Gantt Chart

## 1. Purpose

This document outlines the planned sequence of work for the Medical Chatbot project. It shows how the project moved from planning to implementation, testing, and final documentation.

## 2. Project Phases

| Phase                  | Main Activities                                             | Estimated Duration | Depends On         |
| ---------------------- | ----------------------------------------------------------- | -----------------: | ------------------ |
| Planning               | Define scope, identify goals, gather requirements           |             1 week | None               |
| Analysis and Design    | Draft requirements, architecture, diagrams, and wireframes  |             1 week | Planning           |
| Backend Development    | Build Django views, authentication, chat logic, and storage |            2 weeks | Design             |
| Frontend Development   | Create templates, styling, and JavaScript interactions      |             1 week | Design             |
| Integration            | Connect backend responses with the frontend UI              |             1 week | Backend + Frontend |
| Testing and Refinement | Validate user flows, fix issues, improve usability          |             1 week | Integration        |
| Documentation          | Write report, requirements, design, and user documentation  |             1 week | Testing            |
| Final Packaging        | Organize deliverables and prepare final submission          |             1 week | Documentation      |

## 3. Suggested Timeline

- **Week 1:** project planning and requirements capture
- **Week 2:** design diagrams, architecture, and wireframes
- **Weeks 3-4:** backend implementation and authentication flows
- **Week 5:** frontend styling and interactivity
- **Week 6:** system integration and history handling
- **Week 7:** testing and bug fixing
- **Week 8:** documentation and final PDF preparation

## 4. Task Dependencies

- Requirements must be finalized before detailed design.
- Design must be completed before implementation begins.
- Backend and frontend work must be finished before full integration.
- Integration should be completed before final testing and packaging.

## 5. Task Assignment Note

If the project was completed individually, the same student can be assigned to every task. If the project was completed by a team, each task should be mapped to the responsible member in the final report.

## 6. Final Note

In the final PDF submission, this information should be converted into a visual Gantt chart showing task bars, dependencies, and milestone dates.
